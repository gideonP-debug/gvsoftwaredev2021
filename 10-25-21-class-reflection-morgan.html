Waterfall:

  • Waterfall project management is a tactic that divides the software development life cycle into phases. You can only move to the next phase once the previous one has been completed. This method only occurs once, so each phase is only completed once. Information is gathered as much as possible at the start of a project because any changes after it has started requires you to restart! A couple benefits of using the Waterfall method are that it has straightforward planning and defined team roles. A couple of disadvantages are it's not flexible to uncertainty and testing is only done in later phases of the project.
